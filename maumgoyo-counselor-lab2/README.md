# 마음고요 상담분석실 v11 (20260713v44 — AI 정리본에 5개 분석 통합, 구조화된 보고서 프롬프트, 사이드바 리사이즈)

## 이 버전에서 바뀐 것

### AI 분석용 정리본에 5개 분석 통합 (v44)
"AI 분석용 정리본 만들기"가 기존 6개 섹션에 더해, 5·6순위까지 진행하며 만든 분석을 새 섹션으로 포함하도록 확장했습니다(총 11개 섹션).

```
[6. 개인화된 안전계획 — 효과가 있었던 대처]
[7. 회피 신호 추세]
[8. 관계적 촉발요인]
[9. 변화언어 비율 추세]
[10. 목표 대비 진행률]
[11. 활용 안내] (기존 6번이 뒤로 이동)
```

각 섹션은 화면 카드와 동일한 계산 로직을 공유합니다(`computeSafetyPlanRanked`, `computeAvoidanceTrend`, `aggregateTagHeatmapWithAction`, `computeMotivationTrend`, `computeGoalProgress`로 분리해 화면 카드와 텍스트 정리본이 같은 코드를 재사용하도록 리팩터링).

### 구조화된 임상 보고서 형식 프롬프트 (v44)
정리본 맨 앞의 분석 요청 프롬프트를, 심리치료 사례개념화에서 흔히 쓰는 형식으로 구조화했습니다.

```
1. 사례 개요
2. 심리학적 사례개념화(Case Formulation) — 유발/촉발/지속/보호 요인(4P 모델)
3. 기능분석 — 감정·생각·충동·문제행동의 연쇄관계(관계적 촉발요인 포함)
4. 재발 위험 평가 — 재발 위험·요소, 회피 신호 추세 포함
5. 강점 및 자원 — 실천 효과, 개인화된 안전계획, 변화언어 추세, 목표 진행률
6. 치료적 개입 제안 — 단기/장기 목표로 구분
7. 다음 회기를 위한 제언
```

"확정적 진단명을 붙이지 말 것", "자료에 없는 내용을 지어내지 말 것", "상담자의 최종 판단을 대체하지 않는다는 점을 명시할 것"이라는 조건은 그대로 유지했습니다.

### 사이드바 폭 조정 기능 (v44)
사이드바가 좁아서 상담 메모·AI 분석용 정리본·자기성찰 요약문을 보기 불편하다는 의견을 반영해, 사이드바와 작업 영역 경계에 드래그 가능한 손잡이를 추가했습니다. 기본 폭을 420px로 넓혔고(기존 최대 360px), 280~720px 사이에서 마우스로 자유롭게 조정할 수 있습니다. 화면 폭이 좁으면(1080px 이하) 손잡이는 자동으로 숨겨지고 기존처럼 세로 배치로 전환됩니다.

### 🐛 버그 수정: 변화언어 오탐 (v43)
`detectMotivation()`이 `record.title + record.content`를 함께 검사했는데, "대처/가치실천"이라는 **고정 카테고리 제목 자체에 "가치"라는 변화언어 키워드가 들어있어서**, 내용과 무관하게 대처 관련 기록마다 무조건 "변화언어 후보"로 오탐되고 있었습니다. `record.content`(내담자의 실제 말)만 검사하도록 고쳤습니다. "3. 피드백 준비도"·"9. 회기 피드백 요약" 탭에 모두 영향을 미쳤던 버그입니다.

### 변화언어 비율 추세 + 목표 대비 진행률 (v43)
"3. 피드백 준비도" 탭에 "변화언어 비율 추세" 카드를 추가했습니다. 기존 기능은 지금 몇 건이 변화언어/유지언어로 감지되는지 스냅샷만 보여줬는데, 전반부/후반부로 나눠 **변화언어 비율이 늘고 있는지 줄고 있는지**를 봅니다. 동기강화상담(MI)에서는 변화언어 비율의 증가 자체를 예후 지표로 봅니다.

### 목표 대비 진행률 (v43)
사이드바에 "회복 목표" 패널(충동·문제행동·실천 완료율·삶의 확장감 목표치 입력)을 추가하고, "9. 회기 피드백 요약" 탭에 "목표 대비 진행률" 카드를 추가했습니다. 상담자가 설정한 목표와 이번 기간 실제 평균을 나란히 비교해 "달성/미달"과 그 차이를 보여줍니다. 기존 "위험 기준"(경고 임계값)과는 성격이 달라 별도 패널로 분리했습니다.

임상심리 전문가 관점 재검토의 5·6순위 항목입니다(4순위 시간대/요일 위험 패턴 요약은 보류 중).

### 관계적 촉발요인 전용 카드 (v42)
"5. 연쇄 이해" 탭에 "관계적 촉발요인" 카드를 추가했습니다. `trigger_people`(혼자 있을 때/특정 인물과 함께/갈등 직후/사교 모임 상황)은 기존 촉발단서 히트맵에도 있었지만 장소·시간대·문제행동영역과 섞여 있어 두드러지지 않았습니다. 부부·관계 문제로 상담하는 경우 이 축이 핵심이라 별도 카드로 분리했습니다.

- 기존 히트맵은 평균 충동만 계산했는데, 이 카드는 **평균 문제행동 수준(action_level)도 함께** 계산합니다 — 충동만 높은 것과 실제 행동으로 이어지는 것은 다른 문제라서요.
- 평균 문제행동 수준이 가장 높은 관계적 촉발요인에 대해 실제 상황 문장을 예시로 인용
- STOP/TIPP·개인화된 안전계획과 마찬가지로 환자용 보기에도 그대로 노출됩니다(연쇄 이해 탭은 상담자·내담자가 함께 보는 용도로 설계됨)

임상심리 전문가 관점 재검토 3순위 항목입니다(다음: 시간대/요일 위험 패턴 → 변화언어 비율 추세화 → 목표 대비 진행률).

### 회피 신호 추세 카드 (v41)
"4. 재발 조기경보·위기대처" 탭에 "회피 신호 추세" 카드를 추가했습니다(상담자 전용). 재검토 결과 회피 신호(`avoidance_tags`) 자체는 이미 "5. 연쇄 이해"의 촉발단서 히트맵에 빈도·평균충동으로 들어가 있었지만, **시간에 따른 증가/감소 추세**와 **문제행동과는 별개의 우울·무기력 신호라는 명시적 프레이밍**은 빠져 있어서 그 부분만 새로 추가했습니다.

- 전체 기간을 전반부/후반부로 나눠 하루 평균 회피 신호 개수를 비교
- 후반부에 늘면 "우울·무기력이 진행되고 있을 수 있어 확인이 필요합니다"로 경고 표시(카드색 warn)
- 자주 나온 회피 신호 상위 3개, 회피 신호가 있었던 관찰 건수를 함께 표시
- 문제행동이 줄어도 이 신호가 늘 수 있다는 점을 카드 설명에 명시(듀얼 연속체와 같은 취지)

임상심리 전문가 관점 재검토에서 나온 개선 우선순위 2순위 항목입니다(다음: 관계적 촉발요인 전용 뷰 → 시간대/요일 위험 패턴 → 변화언어 비율 추세화 → 목표 대비 진행률).

### 개인화된 안전계획 카드 (v40)
"4. 재발 조기경보·위기대처" 탭 맨 앞에 "개인화된 안전계획 — 나에게 효과가 있었던 것들" 카드를 추가했습니다. STOP/TIPP 같은 일반 기법과 달리, **이 내담자가 실제로 도움 정도 6점 이상으로 기록한 대처 방법**만 모아서 보여줍니다.

- 같은 대처를 여러 번 기록했으면 평균 도움 점수·기록 횟수·최근 날짜로 묶어서 보여줌
- 평균 도움 점수가 높은 순으로 정렬(최대 5개)
- 기록이 없으면 "아직 없음" 안내 문구로 자연스럽게 처리
- STOP/TIPP과 마찬가지로 **환자용 보기에도 그대로 노출**됩니다(공식 재발신호 카드와 달리 상담자 전용이 아님) — 위기 순간 내담자가 직접 참고할 수 있는 자료이기 때문입니다.

임상심리 전문가 관점에서 앱 전체를 재검토해 발견한 개선 우선순위 중 1순위 항목입니다(2순위 이후: 회피신호 추세 분석, 관계적 촉발요인 전용 뷰, 시간대/요일 위험 패턴, 변화언어 비율 추세화, 목표 대비 진행률 — 순서대로 검토 예정).

### AI 분석용 정리본 상단에 분석 요청 프롬프트 추가 (v39)
"AI 분석용 정리본 만들기" 버튼을 누르면 이제 정리본 텍스트 맨 앞에 분석 요청 프롬프트가 자동으로 포함됩니다. 별도 버튼이 아니라 정리본 자체의 일부입니다. 프롬프트는 다음을 요청합니다.

1) 내담자에 대한 psychological formulation(심리학적 사례개념화)
2) 전문 심리상담사 관점에서 감정·생각·충동·문제행동의 연쇄관계 특성, 재발 위험 및 재발 요소, 작은 실천행동의 효과와 개선점을 분석하고 치료적 개입 내용 제안

기존 6개 섹션(요약 통계/연쇄관계 원자료/재발 위험/실천 효과/확장감/활용 안내) 구조는 그대로 두고, 그 앞에 이 프롬프트만 추가했습니다. 프롬프트 안에도 "자료에 없는 내용을 지어내지 말 것", "이 분석이 상담자의 최종 임상적 판단을 대체하지 않는다는 점을 답변 마지막에 명시할 것"이라는 조건을 넣어, 원 명세서의 "진단·개입 자동결정 금지" 경계가 유지되도록 했습니다.

### "AI 분석용 정리본" 위치 이동 (v38)
사이드바 패널 순서를 "상담 메모" → "AI 분석용 정리본" → "자기성찰 요약문"으로 조정했습니다. 기능 변경 없이 배치만 바뀐 것입니다.

### AI 분석용 정리본 생성 (v37)
별도 명세서(「AI 분석용 정리본 생성」v1.0)에 따라 새 기능을 추가했습니다. 사이드바에 "AI 분석용 정리본 만들기" 버튼을 누르면, 선택된 기간의 `observation`/`practice_definition`/`practice_log`/`daily_checkin` 기록을 6개 고정 섹션으로 정리한 텍스트가 읽기 전용 상자에 나타납니다.

```
[1. 9개 영역별 요약 통계]   — 상황/촉발단서, 생각, 감정, 몸반응, 충동, 문제행동, 대처, 실천, 확장감 각각의 평균·최빈값·추세
[2. 관찰 기록 연쇄관계 원자료] — 날짜순 전체 관찰 기록
[3. 재발 위험 및 요소]      — 정서·인지 재발신호 발생일 수, 문제행동 기록일, 촉발단서별 평균 충동 비교
[4. 작은 실천행동의 효과]   — 실천행동별 수행 횟수, 즐거움/성취감, 예상 대비 실제 차이
[5. 삶의 확장감과 만족도]   — 평균과 전반부/후반부 추세
[6. 활용 안내]              — 진단·개입을 결정하지 않는다는 고정 안내문
```

**경계**: 이 기능은 데이터를 분석하기 좋은 형태로 정리하는 것까지만 합니다. 진단이나 치료 개입을 자동으로 제안하지 않으며, 파일 다운로드가 아니라 화면 텍스트 상자 + 클립보드 복사로만 제공합니다(상담자가 이 텍스트를 별도 AI 챗봇에 붙여넣어 분석을 요청하는 흐름을 전제로 함).

명세서 §6에 제시된 검증 시나리오(촉발단서 3종 모두 "있음 평균 충동 9.0점", 실천 "예상 4.0 대비 실제 8.0 → +4.0", 하루마무리 "3점→6점 → 후반부에 상승 추세")를 그대로 테스트로 재현해 정확히 일치하는 것을 확인했습니다. 실제 21일치 예시 데이터와 빈 데이터(관찰 0건 등)에서도 오류 없이 동작합니다.

### 예시 데이터 대폭 보강 (v36)
기존 3일치(관찰 3건)였던 내장 예시 데이터를 **3주(21일)치, 총 61행**으로 늘렸습니다. 임의의 숫자가 아니라, 다음 논리적 흐름이 뚜렷하게 보이도록 설계했습니다.

- **1주차(초반)**: 충동 6~9점, 문제행동 수준 2~4점으로 높음. 실천기록은 거의 없고, 삶의 확장감·만족도는 2~4점으로 낮음.
- **2주차(전환)**: 충동·문제행동이 서서히 낮아지기 시작. 걷기·감정일기 실천이 등장하고, 확장감도 4~6점대로 오름.
- **3주차(개선)**: 충동 2~3점, 문제행동 수준 0~1점으로 낮아짐. 실천 빈도·즐거움·숙달감이 높고, 확장감은 6~8점대로 상승.
- 즉 **"충동↑ → 문제행동↑"**, **"실천↑ → 확장감↑이면서 문제행동↓"**이라는 두 가지 관계가 그래프(트렌드선, 산점도, 캘린더 히트맵, 나란히 보기 타임라인)에서 실제로 확인됩니다.

추가로 포함된 것:
- 하루에 관찰이 2건인 날(다중 연쇄 처리 확인용), "충동 발생" 모드 3건(충동곡선 시각화용)
- 실천행동 3개 중 1개는 `archived: true`(보관된 실천이 목표치 계산에서 정확히 제외되는지 확인용)
- 예측(prediction) 4건 — `occurred`/`partial`/`did_not_occur`/`pending` 상태를 모두 포함
- `daily_checkin` 21건(매일 1건)
- 촉발단서·문제행동영역에 도박·음주·회피/미루기 등 다양한 값을 섞어 히트맵이 의미 있게 나타나도록 구성

Python 스크립트로 생성한 뒤 실제 파서로 검증했습니다 — 61행 전부 오류 없이 읽히고, 9개 탭 전부 정상 렌더링되며, 보관된 실천행동이 목표치 목록에서 정확히 빠지는 것까지 확인했습니다.

### AI 예비 반성문 초안 프롬프트 추가 (v35)
"자기성찰 요약문" 패널에 두 번째 프롬프트 버튼을 추가했습니다. 기존 "AI 질문 생성용 프롬프트"(질문만 만들게 함)에 이어, **"AI 예비 반성문 초안 프롬프트"**는 자기성찰 요약문을 근거로 예비 수준의 반성문 초안을 만들도록 요청하는 프롬프트입니다.

애초에 이 기능을 만든 이유(근거 없는 형식적 반성문 문제)가 AI 버전으로 재생산되지 않도록, 프롬프트 안에 안전장치를 명시했습니다.

- 이 초안은 최종본이 아니라 **내담자가 자기 언어로 고쳐 쓰기 위한 출발점**이라는 점을 초안 맨 앞에 명시하도록 지시
- 기록에 없는 감정·반성을 지어내지 말 것
- "죄송합니다", "다시는 안 그러겠습니다" 같은 근거 없는 상투적 문구 금지, 그 순간의 생각·감정·충동이 실제로 행동으로 연결된 과정을 구체적으로 서술할 것
- 자기비난만이 아니라 기록에 있는 대처 시도·작은 실천·가치도 균형 있게 포함
- 타인에게 미친 영향 서술은 기록으로 확인 가능한 범위 안에서만, 과장·단정 금지
- 분량 제한(A4 반 페이지 이내)

패널에도 "그대로 제출하지 않도록 안내하세요"라는 문구를 고정으로 표시했습니다.

### 방향 전환: 성찰 질문 자동생성 대신 AI 활용 프롬프트 (v34)
직전 버전에서 시도했던 "규칙 기반으로 성찰 질문 5개 이상을 자동 생성하는 기능"을 제거했습니다. 점수 임계값에 따라 정해진 문장 템플릿에 인용문을 끼워 넣는 방식이라, 여러 날에 걸친 패턴이나 맥락을 실제로 "이해"하지는 못한다고 판단했습니다 — 이 앱이 잘하는 일(자료를 정확하고 빠짐없이 정리하는 것)과 AI가 잘하는 일(그 자료를 읽고 이해해서 질문을 만드는 것)을 억지로 한 곳에서 다 하려던 것이었습니다.

대신 "자기성찰 요약문" 패널에 **"AI 질문 생성용 프롬프트와 함께 복사"** 버튼을 추가했습니다. 누르면 상담자가 실제 AI 도구(Claude 등)에 붙여넣을 수 있는 안내 프롬프트가 자기성찰 요약문 앞에 붙어서 함께 복사됩니다. 프롬프트에는 "구체적 기록에 근거할 것", "무조건 잘못했다는 결론을 유도하지 말 것", "강점도 함께 짚을 것", "질문만 만들고 답은 대신 쓰지 말 것"이라는 조건을 명시했습니다. 원자료만 필요하면 기존 "자료만 복사" 버튼도 그대로 남아 있습니다.

### 자기성찰 요약문 — 날짜 단위 (v33, 변경 없음)
v32의 "성찰 자료(반성문 작성 지원)"를 대체했습니다. 위치와 단위, 형식을 모두 바꿨습니다.

- **위치**: "5. 연쇄 이해" 탭이 아니라 **사이드바의 "상담 메모" 바로 아래**(상담자 전용 영역이라 개수가 늘어나도 부담 없음)
- **단위**: 문제행동이 있었던 연쇄가 아니라 **기록이 있는 날짜마다 하나씩**. 관찰·실천기록·확장감 체크인 중 하나라도 있는 날짜 기준이며, **최대 14개**로 제한합니다(가장 최근 날짜부터).
- **형식**: 질문이 딸린 카드가 아니라, **AI 도구 등에 그대로 넣어 쓸 수 있는 정리된 평문 텍스트**로 바꿨습니다. 상담자가 이 텍스트를 원자료 삼아 성찰문 작성을 돕는 워크플로를 염두에 둔 설계입니다.
- 각 날짜 텍스트는 그날의 상황·촉발단서·생각·감정·몸반응·충동곡선·행동·대처·(스스로 적은 가치와 알아차림)에 더해, **"마음-행동 관계" 한 줄 요약**(예: "충동이 높아지며 결국 행동으로 이어짐" / "충동은 높았지만 행동으로 이어지지 않음(대처가 작동함)")을 자동으로 붙입니다. 같은 날 실천기록·확장감 체크인·걱정-결과 비교가 있으면 함께 포함됩니다.
- 날짜별 "복사" 버튼과 전체를 한 번에 합쳐 복사하는 "전체 복사" 버튼 제공
- 이 기능은 반성문을 대신 작성하지 않습니다 — 사실을 정리한 원자료만 만들고, 실제 문장 작성은 상담자·내담자의 몫입니다.

### 성찰 자료(반성문 작성 지원) — v32 초판 (연쇄 단위, 이후 v33에서 날짜 단위로 대체됨)
"5. 연쇄 이해" 탭에 "성찰 자료(반성문 작성 지원)" 컬럼을 추가했습니다(상담자 전용). `action_level ≥ 1`인(실제로 문제행동이 있었던) 모든 기록마다 자동으로 카드가 생성됩니다.

**이 기능은 반성문을 대신 작성하지 않습니다.** 근거 없이 "무조건 잘못했다"는 형식적 반성문은 재범 위험 감소에 기여하지 못한다는 문제의식(치료 문헌의 "programmatic correctness" 현상과 일치)을 반영해, 다음만 제공합니다.

- **사실 기술**: 상황, 촉발단서, 관련 문제행동 영역, 실제로 한 행동 — 판단 없이 그대로
- **그 순간의 내적 상태**: 생각·감정·충동(곡선 포함)·몸 반응을 점수와 함께, 내담자 본인의 문장 그대로 인용
- **이미 하고 있는 노력**: 같은 기록에 적힌 가치·다음 작은 행동·알아차림을 함께 표시해, 순수 자기비난이 아니라 책임과 강점을 같이 다루도록 함(Good Lives Model 방향)
- **성찰 질문 5개** (답은 제공하지 않음): 놓친 신호, 그 생각이 사실이었는지, 이전 대처방법과의 연결, **주변 사람에게 미친 영향**(기본 포함, 사례에 안 맞으면 상담자가 생략 안내), 다음에 다르게 할 일
- 카드 상단에 "내담자가 본인의 언어로 직접 작성해야 한다"는 고정 안내문

이 기능은 성범죄뿐 아니라 이 앱이 다루는 모든 문제행동(도박·음주·분노표출 등)에 공통으로 적용됩니다. 주변 영향 질문은 관계적 영향이 없는 사례에서는 상담자가 읽고 넘기도록 안내 문구만 달아두었고, 별도의 켜기/끄기 UI는 만들지 않았습니다(코드 복잡도 대비 실익이 적다고 판단).

### CSV 연동 명세서 2.0 반영 (v31)
`마음고요 관찰과 실천`의 CSV 구조가 정리되면서(문서 버전 2.0, schema_version은 `maeumgoyo_csv_v1` 유지) 여러 필드의 **타입**이 바뀌었습니다. 이전 버전과의 호환을 고려하지 않고 새 구조 기준으로 다시 맞췄습니다.

- **`archived`가 모든 record_type에서 문자열 `"0"/"1"`이 아니라 진짜 JSON 불리언(`true`/`false`)으로 통일**됨에 따라, 판정 로직을 `payload.archived === true`로 단순화
- **`practice_definition`은 `archived: true`여도 `counselor_full`에 "참조용으로 항상 포함"됨을 반영**: 보관(중단)된 실천 정의는 목표치·완료율 계산(`practicePlans`)에서는 제외하되, 그 실천에 달린 `practice_log`는 원본 행에서 이름을 직접 찾아오므로 계속 정상적으로 그룹 표시됩니다. `observation`/`practice_log`/`prediction`/`daily_checkin`은 `archived: true`면 record_type과 share_mode에 관계없이 분석에서 제외합니다.
- `emotion_custom`/`body_custom`이 문자열에서 배열로 바뀌었지만, 기존 `textList()`/`joinText()` 헬퍼가 배열을 이미 처리하고 있어 별도 수정 없이 정상 작동함을 확인
- `practice_log`의 파생 필드 `practice_score`가 규격에서 제거됨에 따라, 관련 하위호환 fallback 코드를 삭제(`pleasure_score`/`mastery_score`만 사용)
- `daily_checkin`이 같은 날짜에 2건 이상 있으면("하루 1건 원칙" 위반) 경고를 표시하도록 검증 추가
- 내장 예시 CSV("예시" 버튼)를 2.0 규격 그대로 다시 만들어, 보관된 실천행동과 `daily_checkin` 2건을 포함하도록 갱신

### 🐛 버그 수정: 빈 문자열 선택 필드가 0으로 잘못 읽히던 문제 (v31)
실제 샘플 CSV로 테스트하던 중 발견했습니다. `urge_initial_score`/`urge_end_score`/`actual_severity`/`expected_pleasure_score` 같은 **선택 입력** 숫자 필드가 CSV에서 빈 문자열(`""`)로 오면, JavaScript의 `Number("") === 0`라는 특성 때문에 "기록 없음"이 아니라 **"0점"으로 잘못 해석**되고 있었습니다. `score10()`/`score5()`/`wholeNumber()`가 빈 문자열·공백·`null`·`undefined`를 먼저 걸러내고 나서 숫자로 변환하도록 고쳤습니다. 실제 샘플 CSV(`obs-0002`의 빈 충동곡선 값, `pred-0002`의 미확인 `actual_severity`)로 이 버그가 재현되고 수정되는 것을 확인했습니다.

### 전체 백업(backup_full)/range_days 관련 (v29~v30)
`backup_full`이 실제로 어떤 `range_days` 값을 담는지 확인되지 않아, **어떤 값이 와도 신뢰하지 않도록** 만들었습니다. "선택한 공유 기간과 무관하게 저장된다"는 설명 자체가 range_days는 backup_full의 실제 내용을 설명할 수 없다는 뜻이기 때문입니다(값이 잘못돼서가 아니라 개념적으로 적용 불가). `resolveRangeWindow()`가 `share_mode=backup_full` 행이 하나라도 있으면 range_days 값과 무관하게 항상 실제 관측된 관찰 날짜의 최소~최대 범위를 사용하도록 고쳤습니다. 이 값을 기준으로 계산되는 재발신호·실천계획 완료율도 함께 안전해집니다. `counselor_full`은 기존처럼 range_days를 그대로 신뢰합니다.

### 전체 백업(backup_full) CSV 읽기 지원 (v29)
`마음고요 관찰과 실천`의 "공유하기 → 숨긴 기록 포함 전체 CSV 백업" 버튼으로 만들어지는 파일을 읽을 수 있습니다. 이 파일은 "선택한 공유 기간과 무관하게 모든 기록과 보관 상태를 저장"하는, 기존 `counselor_full`(기간 한정 공유본)과는 성격이 다른 파일입니다.

- `share_mode`가 `counselor_full` 또는 `backup_full`인 행을 분석 대상으로 인식
- `backup_full`에서 `payload_json.archived`가 `"1"` 또는 `true`인 기록(보관 처리된 기록)은 record_type과 무관하게 분석에서 제외
- 공유 모드 표시가 "전체 백업본(backup_full, 보관 기록 제외 분석)"과 "상담자용 상세(counselor_full)"를 구분해서 보여줌
- **전체 백업은 기간 제한이 없어 파일이 클 수 있다는 점을 반영**: 파일 크기 상한을 5MB→20MB로 올리고, "필요한 기간으로 좁히세요" 같은 부정확한 안내 문구를 제거. 5,000행 초과 시에도 backup_full이면 "정상적으로 클 수 있음"으로, 그 외에는 기존처럼 "이어붙임 의심" 경고로 구분
- `counselor_full`과 `backup_full`이 한 파일에 섞여 있으면 경고(실제 내보내기에서는 두 값이 각각 다른 버튼으로 별도 파일을 만들므로, 섞여 있으면 파일이 잘못 합쳐졌을 가능성이 있음)

### 환자용 보기 상태 표시 방식 변경 (v27)
환자용 보기로 전환하면 화면 상단에 "환자용 보기 중 · 상담 메모·CSV 관리·분석 도구는 숨겨져 있습니다"라는 배너가 떴는데, 이 문구를 제거했습니다. "무언가를 숨기고 있다"는 문구를 환자가 직접 보게 하는 것은 불필요할뿐더러, 오히려 불신을 키울 위험이 있다고 판단했습니다. 지금은 헤더의 토글 버튼 라벨("환자용 보기" ↔ "상담자용 보기로 전환")만으로 상담자가 현재 상태를 확인합니다.

- "연쇄 기록 전체 보기" 섹션(모든 연쇄의 원문 텍스트 목록)
- 1번 탭(자료 확인): CSV 구조·메타데이터 확인용이라 전체를 상담자 전용으로 처리, 환자용 보기에서는 안내 문구만 표시
- 4번 탭의 "공식 재발신호" 카드(조건 X/6 충족 같은 원점수) — STOP/TIPP과 나머지 위기대처 안내는 그대로 노출됨

**그대로 보이는 것 (환자용)**
- 2, 3, 5, 6, 7, 8번 탭 전체
- **9번 탭(회기 피드백 요약)도 전체 노출** — 상담자와 논의 후 "환자와 같이 보는 게 낫다"고 결정

**설계 원칙**: 새 카드나 그래프를 추가할 때 매번 "이건 환자용?"을 결정하지 않아도 되도록, 기본값은 "보임"으로 두고 소수의 민감한 요소에만 `counselor-only` 클래스를 붙였습니다. CSS 한 줄(`body.patient-mode .counselor-only { display: none; }`)로 처리되며, 순수 표시 전환이라 토글 시 데이터 재계산이나 재렌더링이 필요 없습니다.

## 이전 버전 변경 이력

### 버그 수정: "실천" 줄이 비어 보이던 문제 (v25)
`buildDualAxisDays()`가 그래프에 표시할 날짜 목록을 만들 때 observation과 daily_checkin의 날짜만 모으고, practice_log의 날짜는 포함하지 않았습니다. 그래서 그날 관찰기록이나 확장감 체크인 없이 실천만 기록한 날은 그래프에 칸 자체가 생기지 않아 초록 점이 통째로 사라져 보였습니다. 실천 기록 날짜도 날짜 목록에 포함하도록 고쳤습니다.

### 날짜 라벨 가독성 수정 (v24)
날짜 눈금 라벨(예: `07-05`)이 칸 너비(28px)에 거의 꽉 차는 크기라, 연속된 날짜 데이터가 있으면 옆 라벨과 맞닿아 마치 숫자가 이어붙은 것처럼 보이는 문제가 있었습니다. 라벨 표시 간격에 최소 2칸(56px) 여백을 두고, 날짜축이 세 줄 전체의 공통 축임을 보여주는 구분선과 "날짜" 라벨을 추가했습니다.

### CSV 연동 명세서 v1.1 반영 — `daily_checkin` (v23)
신규 record_type `daily_checkin`(삶의 확장감·만족도)을 CSV 읽기 계층에 추가했습니다. 문제행동(부정 축)과 확장감(긍정 축)을 절대 하나의 그래프로 합치지 않고 항상 나란히 배치하는 듀얼 연속체 모델 원칙을 반영했습니다. "2. 오늘의 측정 피드백" 탭에 "문제행동과 삶의 확장감, 나란히 보기"와 "문제행동×확장감 산점도" 두 그래프를 추가했습니다.

### 감정×충동 산점도 · 문제행동 캘린더 히트맵 (v22)
"2. 오늘의 측정 피드백" 탭에 감정×충동 산점도와 문제행동 캘린더 히트맵을 추가했습니다. 기준일은 시스템 날짜가 아니라 CSV 안 가장 최근 observation 날짜를 사용합니다.

### 버그 수정: "8. 걱정-결과 비교" 버튼 무반응 (v21)
`renderIntro()`의 안내문구 목록에 `prediction` 탭 항목이 누락되어 있던 문제를 고쳤습니다.

### 촉발단서 히트맵 · 충동곡선 시각화, 요약저장·인쇄 폴리싱 (v19~v20)
"연쇄 이해" 탭에 촉발단서/문제행동영역별 히트맵을 추가했고, "요약 저장"/"요약 복사"에 그동안 추가된 모든 신규 기능(공식 재발신호, 예측 통계, 즐거움/숙달감 평균)을 반영했습니다.

### 걱정-결과 비교 탭 + 즐거움·숙달감 비교 (v17~v18)
`prediction`과 `pleasure_score`/`mastery_score`/`expected_*`를 전용 화면으로 노출했습니다.

### CSV 읽기 + 공식 재발신호 + 죽은 코드 정리 (v15~v16)
`maeumgoyo_csv_v1` 전용 단일 읽기 경로로 재작성, 연동명세서 §5-2 3일 슬라이딩 윈도우 재발신호 구현, 미사용 코드 제거.
