# 마음고요 상담분석실 v11 (20260710v27 — 환자용 보기 안내 배너 제거)

## 이 버전에서 바뀐 것

### 환자용 보기 상태 표시 방식 변경 (v27)
환자용 보기로 전환하면 화면 상단에 "환자용 보기 중 · 상담 메모·CSV 관리·분석 도구는 숨겨져 있습니다"라는 배너가 떴는데, 이 문구를 제거했습니다. "무언가를 숨기고 있다"는 문구를 환자가 직접 보게 하는 것은 불필요할뿐더러, 오히려 불신을 키울 위험이 있다고 판단했습니다. 지금은 헤더의 토글 버튼 라벨("환자용 보기" ↔ "상담자용 보기로 전환")만으로 상담자가 현재 상태를 확인합니다.

### 환자용 보기 토글 (v26)
헤더에 "환자용 보기" 버튼을 추가했습니다. 누르면 상담 메모·CSV 관리 도구·위험 기준 조정 등 상담자 전용 요소가 화면에서 사라지고, 환자와 함께 봐도 되는 내용만 남습니다. 다시 누르면 상담자용 화면으로 즉시 돌아옵니다(확인 팝업 없음).

**모니터가 2대인 PC를 자동으로 감지해서 새 창을 띄우는 방식은 채택하지 않았습니다.** 이런 방식은 브라우저 제한(크로미움 계열만 지원), 보안 컨텍스트 제약(`file://`로 여는 지금 배포 방식과 충돌 가능), 두 창 간 상태 동기화라는 관리 비용이 있습니다. 대신 **창은 하나만 유지하고, 그 안에서 표시 모드만 전환**하는 방식을 택했습니다. 모니터 2대를 실제로 활용하고 싶으면 이 창을 상담자가 직접 두 번째 모니터로 드래그하면 됩니다 — OS가 이미 잘하는 일을 앱이 대신 하지 않습니다.

**숨겨지는 것 (상담자 전용)**
- 사이드바 전체: CSV 불러오기, 위험 기준 입력, 자료 범위, **상담 메모**
- 헤더 버튼: 예시, 새로고침, 자료 제거, 인쇄, 요약 저장
- "연쇄 기록 전체 보기" 섹션(모든 연쇄의 원문 텍스트 목록)
- 1번 탭(자료 확인): CSV 구조·메타데이터 확인용이라 전체를 상담자 전용으로 처리, 환자용 보기에서는 안내 문구만 표시
- 4번 탭의 "공식 재발신호" 카드(조건 X/6 충족 같은 원점수) — STOP/TIPP과 나머지 위기대처 안내는 그대로 노출됨

**그대로 보이는 것 (환자용)**
- 2, 3, 5, 6, 7, 8번 탭 전체
- **9번 탭(회기 피드백 요약)도 전체 노출** — 상담자와 논의 후 "환자와 같이 보는 게 낫다"고 결정

**설계 원칙**: 새 카드나 그래프를 추가할 때 매번 "이건 환자용?"을 결정하지 않아도 되도록, 기본값은 "보임"으로 두고 소수의 민감한 요소에만 `counselor-only` 클래스를 붙였습니다. CSS 한 줄(`body.patient-mode .counselor-only { display: none; }`)로 처리되며, 순수 표시 전환이라 토글 시 데이터 재계산이나 재렌더링이 필요 없습니다.

## 이전 버전 변경 이력

### 버그 수정: "실천" 줄이 비어 보이던 문제 (v25)
`buildDualAxisDays()`가 그래프에 표시할 날짜 목록을 만들 때 observation과 daily_checkin의 날짜만 모으고, practice_log의 날짜는 포함하지 않았습니다. 그래서 그날 관찰기록이나 확장감 체크인 없이 실천만 기록한 날은 그래프에 칸 자체가 생기지 않아 초록 점이 통째로 사라져 보였습니다. 실천 기록 날짜도 날짜 목록에 포함하도록 고쳤습니다.

### 날짜 라벨 가독성 수정 (v24)
날짜 눈금 라벨(예: `07-05`)이 칸 너비(28px)에 거의 꽉 차는 크기라, 연속된 날짜 데이터가 있으면 옆 라벨과 맞닿아 마치 숫자가 이어붙은 것처럼 보이는 문제가 있었습니다. 라벨 표시 간격에 최소 2칸(56px) 여백을 두고, 날짜축이 세 줄 전체의 공통 축임을 보여주는 구분선과 "날짜" 라벨을 추가했습니다.

### CSV 연동 명세서 v1.1 반영 — `daily_checkin` (v23)
신규 record_type `daily_checkin`(삶의 확장감·만족도)을 CSV 읽기 계층에 추가했습니다. 문제행동(부정 축)과 확장감(긍정 축)을 절대 하나의 그래프로 합치지 않고 항상 나란히 배치하는 듀얼 연속체 모델 원칙을 반영했습니다. "2. 오늘의 측정 피드백" 탭에 "문제행동과 삶의 확장감, 나란히 보기"와 "문제행동×확장감 산점도" 두 그래프를 추가했습니다.

### 감정×충동 산점도 · 문제행동 캘린더 히트맵 (v22)
"2. 오늘의 측정 피드백" 탭에 감정×충동 산점도와 문제행동 캘린더 히트맵을 추가했습니다. 기준일은 시스템 날짜가 아니라 CSV 안 가장 최근 observation 날짜를 사용합니다.

### 버그 수정: "8. 걱정-결과 비교" 버튼 무반응 (v21)
`renderIntro()`의 안내문구 목록에 `prediction` 탭 항목이 누락되어 있던 문제를 고쳤습니다.

### 촉발단서 히트맵 · 충동곡선 시각화, 요약저장·인쇄 폴리싱 (v19~v20)
"연쇄 이해" 탭에 촉발단서/문제행동영역별 히트맵을 추가했고, "요약 저장"/"요약 복사"에 그동안 추가된 모든 신규 기능(공식 재발신호, 예측 통계, 즐거움/숙달감 평균)을 반영했습니다.

### 걱정-결과 비교 탭 + 즐거움·숙달감 비교 (v17~v18)
`prediction`과 `pleasure_score`/`mastery_score`/`expected_*`를 전용 화면으로 노출했습니다.

### CSV 읽기 + 공식 재발신호 + 죽은 코드 정리 (v15~v16)
`maeumgoyo_csv_v1` 전용 단일 읽기 경로로 재작성, 연동명세서 §5-2 3일 슬라이딩 윈도우 재발신호 구현, 미사용 코드 제거.
