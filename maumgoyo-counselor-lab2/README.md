- 같은 `id`가 반복되면 `updated_at`이 더 최신인 기록만 분석에 반영합니다.

- 누적 자료에 `counselor_full`과 `backup_full`이 함께 들어오면 분석 범위가 넓어질 수 있음을 경고합니다.
- `2. 오늘의 측정 피드백`에는 주별 변화 피드백 그래프와 주별 통계표가 포함되어 있습니다.

## 사용 방법

`index.html`을 브라우저로 열고, 마음고요 관찰과 실천 앱에서 내보낸 CSV를 불러오면 됩니다. 같은 내담자의 주차별 CSV를 순서대로 불러오면 자료가 누적됩니다. 자료를 새로 시작하려면 `자료 제거`를 먼저 누르세요.

## 개인정보 안내

이 앱은 CSV를 서버에 저장하지 않습니다. 다만 브라우저 탭 안에서는 불러온 원본 기록이 분석을 위해 유지됩니다. `전체 자료 저장(CSV)`으로 내보낸 파일에는 민감한 원문 기록이 포함될 수 있으므로 승인된 보관 위치에 저장하세요.





## 20260716v65 저장 기능 정리

- `상담기록 요약 저장` / `상담기록·비교 요약 저장(JSON)`: 현재 회기 기록 보관과 다음 회기 비교용 JSON입니다. 원본 CSV 전체가 아니라 분석 요약, 핵심 수치, 상담 메모, 연쇄 요약을 저장합니다.
- `AI 회기준비 자료 저장(MD)`: 최근 1~2주 핵심 정보, 프로젝트 연속성 프롬프트, 이전 회기 맥락 입력란을 마크다운으로 저장합니다. 이번 50분 회기의 초점, 작업 순서, 질문 예시를 구체화할 때 쓰는 준비 자료입니다.
- `AI 상세분석 원자료 저장(TXT)`: 관찰·실천 기록을 길게 정리한 원자료입니다. 회기 프로토콜보다 넓은 자료 검토나 추가 분석에 사용합니다.
- `누적 원본자료 저장(CSV)`: 여러 번 불러온 내담자 CSV를 하나의 원본 CSV로 다시 묶어 저장합니다. 실제 기록 문장이 담긴 민감정보 파일입니다.


## 20260716v65 별칭·AI 연속성 업데이트

- CSV의 `client_alias`가 비어 있으면 상담분석실이 `MG-UNKNOWN-YYYYMMDD-XXXX` 형식의 임시 비식별 자료 코드를 부여합니다.
- 같은 탭에서 이어서 불러오는 빈 별칭 CSV에는 같은 임시 코드를 적용하여 병합 안정성을 높입니다.
- 일부 행만 `client_alias`가 비어 있고 파일 안에 단일 별칭이 있으면, 비어 있는 행에 그 별칭을 보완합니다.
- `AI 회기준비 자료 저장`은 `.md` 마크다운 파일로 저장되며, 상단에 프로젝트 연속성 프롬프트와 이전 회기 맥락 입력란이 포함됩니다.
- AI 전달자료 이름표는 `마음고요-{client_alias}` 형식으로 자동 제안됩니다.


## 20260718v84 AI 전달용 회기연결 메모 정리

- 상담사가 `AI 전달자료 이름표`를 직접 수정할 수 있습니다. 수정된 이름은 AI 회기준비 자료와 AI 전달용 회기연결 메모 파일명에 반영됩니다.
- `이전 회기연결 메모 불러오기` 기능은 지난 회기에 저장한 `*_AI전달용_회기연결메모.md` 파일을 새 AI 회기준비 자료에 포함합니다.
- `AI 전달용 회기연결 메모 저장(MD)` 기능은 상담자가 채택한 AI 제안, 보류한 가설, 다음 회기 확인 질문, 다음 AI에게 전달할 맥락을 구조화해 저장합니다.
- 이 기능은 AI의 자동 메모리가 아니라, 상담자가 외부 AI에 다음 회기에도 이어서 참고시키고 싶은 치료 맥락을 직접 관리하는 연결 메모입니다.


## 20260718v84 저장 이름표 설명 보강

- `내담자 폴더/파일 표시명`은 일반 저장파일의 기본 이름표입니다. 상담기록 요약, 누적 원본자료, 표준 근거자료, 프롬프트 저장, 이번 회기 비교용 요약 파일명에 주로 반영됩니다.
- `AI 전달자료 이름표`는 AI 분석을 이어가기 위한 이름표입니다. AI 표준 근거자료의 `project_name`, AI 회기준비 자료의 프로젝트 연속성 문구, AI 전달용 회기연결 메모의 파일명과 본문에 반영됩니다.
- 두 이름은 같게 써도 되지만, 구분하려면 `홍길동` / `홍길동_AI회기연결`처럼 적으면 됩니다.


## 20260716v65 저장 피드백 개선

- 각 저장 카드 설명에 해당 파일의 실제 활용 용도를 추가했습니다.
- 저장 버튼을 누르면 앱 화면에 `저장 요청 완료: 파일명` 메시지가 남습니다.
- 저장 직후 버튼이 잠시 비활성화되어 반복 클릭을 줄입니다.
- 저장 파일은 브라우저 다운로드 방식으로 내려갑니다. 내담자별 폴더를 고르려면 브라우저 설정에서 `다운로드 전에 각 파일의 저장 위치 확인`을 켜는 방식을 권장합니다.
- 앱이 직접 폴더 선택창을 여는 방식은 현재 실행 환경에서 앱이 닫힐 수 있어 사용하지 않습니다.
