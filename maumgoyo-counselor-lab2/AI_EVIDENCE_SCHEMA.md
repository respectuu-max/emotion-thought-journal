# 마음고요 AI 표준 근거자료 v1

## 목적

`maeumgoyo_ai_evidence_v1`은 상담자용 앱의 여러 AI 출력물을 하나의 공통 근거자료로 통합하기 위한 JSON 규격이다. 이 파일은 보고서가 아니며 특정 보고서를 작성하라는 프롬프트도 포함하지 않는다. 회기준비, 상세분석, 자기성찰 보조, 주간 치료참여 및 재범방지 실천기록 등은 별도의 목적별 프롬프트가 이 파일을 읽어 생성한다.

누적 원본 CSV는 자료 보관과 재불러오기를 위해 계속 별도로 유지한다.

## 목적별 프롬프트 프로필

동일한 표준 JSON은 A 회기준비, B 상세 사례분석, C 다음 회기 임상맥락, D 자기성찰 질문, E 예비 자기성찰문, F 주간 치료참여 및 재범방지 실천기록, G 월간 치료과정 요약, H 상담자 의견서 근거개요에 공통으로 사용한다. 프로필은 자료를 바꾸지 않고 결과물의 목적과 형식만 지정한다.

## 출처 코드

| 코드 | 의미 | 외부 문서에서의 취급 |
|---|---|---|
| `SR` | 내담자 자기보고 또는 내담자 앱 입력 | 독립적으로 확인된 사실과 구분한다. |
| `AM` | 앱이 원자료에서 계산한 수치·연쇄·요약 | 상담자의 검토가 필요한 보조정보이다. |
| `CO` | 상담자가 직접 입력한 관찰·확인 사실 | 직접 확인 범위와 한계를 함께 기술한다. |
| `AI` | AI가 생성했으나 채택되지 않은 해석 | 사실처럼 사용하지 않는다. |
| `CH` | 상담자가 검토하여 채택한 AI 제안·해석 | 상담자의 판단임을 표시하고 원자료와 대조한다. |

| `period_and_completeness` | 최초·최종 기록일, 기록일, 관찰기간 내부 무기록일, 지연 기록 |

| `source_evidence` | CSV 행별 원자료와 `SR-*` 근거 ID |
| `counselor_evidence` | 상담 메모·과제·직접 확인·확인 한계와 `CO-*` 근거 ID |
| `clinical_continuity` | 이전 맥락, 채택·보류 제안, 다음 회기 질문 |
| `limitations` | 자기보고, 무기록일, 앱 계산, 외부 제출에 관한 해석 한계 |
| `validation` | 검증기 버전, 상태, 오류·경고 수, 오류·경고 내용과 개별 점검 결과 |

## 감사 기록 (gratitude_reflection)

`daily_checkin`의 `gratitude_others`(타인에 대한 감사), `gratitude_self`(자신에 대한 감사)는 `expansion_score`(삶의 경험 확장/만족도)와 같은 카드에서 나오는 자료이며, `current_period_app_measures.gratitude_reflection`(provenance `AM`)에 다음과 같이 집계된다.

| 필드 | 내용 |
|---|---|
| `others_count` / `self_count` | 이번 기간 전체 감사 건수 |
| `days_with_any_gratitude` | 감사를 1건 이상 기록한 날 수 |
| `compared_to_previous_snapshot` | 직전 회기 스냅샷 대비 건수 차이(단순 수치. 개선/악화 라벨 없음) |
| `representative_entries` | 최신순으로 선정한 대표 원문과 `SR-*` 근거 ID(타인/자신 구분) |

날짜별 원문은 `daily_reflection_summaries`에도 그날의 삶의 경험 확장/만족도 옆에 함께 표시된다.

이 항목은 선택 입력이므로 다음을 지킨다.

- 건수·존재 여부만으로 반성의 진정성, 정서 개선, 치료 성실도를 판단하지 않는다.
- `comparison`이나 회기비교 화면의 "개선/악화" 자동판정 대상에 포함하지 않는다.
- 대표 원문 선정은 최신순일 뿐이며 내용에 대한 긍정·부정 평가를 포함하지 않는다.

## 4번(개입 분기 판단) 근거 (intervention_branch_rationale)

9단계 치료작업 중 4번(개입 분기 판단)만 특정 record_type에 대응하지 않는 순수 임상판단이다.
`treatment_participation.intervention_branch_rationale`(provenance `CO`, 근거 ID `CO-BRANCH-001`)에
상담자가 그 회기에 CBT/DBT/ACT/행동활성화 중 무엇을 택했고 왜 그랬는지 짧게 남긴다. 선택 입력이며,
비어 있어도 오류나 경고가 되지 않는다. 다만 비어 있으면 원자료만으로 "왜 이 개입을 택했는가"를
재구성할 수 없다는 한계가 `limitations`에 명시된다.

## 자살·자해 위험 선별 (safety_screening)

표준화된 진단·평가 도구가 아니라, 상담자가 **매 회기 직접 확인**하는 3단계 선별이다(provenance CO).
재발위기 자동 초점 판정(`automatic_focus`)과 완전히 별개이며, 위험도가 "high_risk"로 표시되면
화면 최상단에 자동 초점 판정과 무관한 별도 배너로 강제 표시된다.

| 필드 | 내용 |
|---|---|
| `status`/`status_label` | 기록하지 않음 / 없음 / 경미한 사고 있으나 계획·의도 없음 / 위험도 높음 |
| `note_evidence_id` | 상담자 메모가 있으면 `CO-SAFETY-001` 참조 |
| `persists_across_sessions` | 항상 `false` — 이전 회기 값이 자동으로 이어지지 않고 매 회기 새로 확인해야 함 |

**절대 원칙**: 이 항목은 **어떤 목적의 문서(A~H)에도 절대 포함하지 않는다.** 모든 프롬프트 프로필의
공통 규칙에 "JSON에 safety_screening 필드가 있어도 절대 인용·요약하지 마라"는 지침이 들어가 있다.
이 항목은 세션 내 안전 확인용이며, 법원 제출 등 외부 문서화 목적과는 완전히 분리되어 있다.

`relational_impact.disclosure_status`가 `discovered`(발각됨)로 바뀌면, 배신외상·재정위기 문헌에서
급성 자살위험이 몰리는 시기로 보고되는 지점이라 화면에서 안전 선별을 다시 확인하도록 상기시킨다.

## 관계적 영향 (relational_impact)

문제행동이 배우자·가족 등과의 신뢰에 준 영향과 회복 과정을 추적하는 선택 입력 축이다. 성중독뿐
아니라 도박·알코올 등 가족의 신뢰 손상이 흔한 모든 사례에 공통으로 쓴다(provenance `CO`).

| 필드 | 내용 |
|---|---|
| `disclosure_status`/`disclosure_status_label` | 기록하지 않음 / 해당사항 없음 / 비공개 / 부분 공개 / 공개함 / 발각됨 |
| `repair_status`/`repair_status_label` | 기록하지 않음 / 해당사항 없음 / 회복 시도 전 / 회복 진행 중 / 안정화되어 감 |
| `counselor_note_evidence_id` | 상담자 관찰 메모가 있으면 `CO-RELATIONAL-NOTE-001` 참조 |

**반드시 지킬 것**: 이 항목은 상담자의 관찰이며 내담자 자기보고나 독립적 확인을 대체하지 않는다.
상태 구분(범주)만 구조화되어 있고, 자유 텍스트 메모에도 배우자·가족 등 **제3자의 신원을 짐작할 수
있는 정보(실명, 직장, 고유한 특징 등)는 절대 포함되어서는 안 된다.** 외부 제출 문서(F/G/H 프로필)에는
상태 라벨만 사실로 제시하고, 자유 텍스트 메모를 그대로 인용하지 않는다.

## 개인별 임계값 보정 (threshold_profile)

위험 연쇄·정서 주의·인지 주의 등 여러 지표는 정서(emotion)·인지충동(cognitiveUrge)·충동(urge)
임계값을 기준으로 계산된다. 이 임계값은 전역 기본값(정서 7, 인지주의 6, 충동 8, 문제행동 4)이거나,
이 내담자 자신의 지난 자기보고 분포에서 계산한 개인화 값일 수 있다. `current_period_app_measures.threshold_profile`
(provenance `AM`)에 다음을 남긴다.

| 필드 | 내용 |
|---|---|
| `personalized` | 개인화 임계값이 적용됐는지 여부 |
| `active_thresholds` | 현재 실제로 쓰이고 있는 임계값 |
| `generic_defaults` | 전역 기본값(비교 기준) |
| `calibration` | 적용됐다면 계산 방식(백분위수), 적용일, 근거 자료 건수·중앙값·75백분위 |

계산 방식과 안전장치:

- 정서·충동은 이 내담자 자신의 자기보고 분포에서 **75백분위**를, 인지 주의는 같은 충동 분포의
  **중앙값(50백분위)**을 제안값으로 계산한다(원래도 인지 주의가 충동보다 낮은, 더 이른 경고선이기 때문).
- 관찰 기록이 10건 미만이거나, 값이 대부분 비슷해 변별력이 없으면(사분위범위 1점 이하) 개인화를
  시도하지 않고 전역 기준을 유지한다.
- **절대 자동으로 적용되지 않는다.** 상담자가 화면에서 제안값을 확인하고 명시적으로 적용해야 하며,
  적용 이력은 다음 회기 요약 파일을 통해 이어진다.
- **문제행동수준(action) 임계값은 개인화하지 않는다.** 이건 자기보고 강도가 아니라 실제 행동화
  여부를 판정하는 안전망이기 때문이다.
- 개인화가 4주 이상 지나면 재보정을 권장하는 안내가 뜬다(자동으로 재계산되지는 않는다).
- 개인화된 임계값을 쓴 문서는 **다른 내담자의 문서나 일반 기준과 수치를 직접 비교할 수 없다.**
  F/G 프로필처럼 외부에 제출되는 문서에는 이 사실이 함께 명시되어야 한다.

## 프로필 F 작성에 필요한 최소 조건

법원 제출용 「주간 치료참여 및 재범방지 실천기록」 초안을 만들기 전 다음을 확인한다.

1. `treatment_participation.session_date`와 `attendance_status`가 입력되어 있다.
2. 상담과제가 있다면 `assigned_tasks`와 `CO-TASK-*` 근거가 존재한다.
3. 상담자가 직접 확인한 사실과 내담자의 일상 자기보고가 구분되어 있다.
4. `period_and_completeness`의 무기록일과 지연 기록을 숨기지 않는다.
5. 구체적인 실천은 반드시 `SR-*` 또는 `CO-*` 근거 ID와 연결한다.
6. 계획, 실제 수행, 부분 수행, 미수행을 서로 구분한다.
7. 개인정보 후보 경고가 없더라도 실명·기관명·구체적 장소를 사람이 다시 확인한다.

## 해석 제한

- 기록의 존재나 빈도는 반성의 진정성을 증명하지 않는다.
- 수치가 개선되어도 치료효과나 재범위험 감소를 자동으로 결론내리지 않는다.
- 감사 기록(`gratitude_others`/`gratitude_self`)은 선택 입력이며, 건수·기록일 증감만으로 정서 개선이나 치료 성실도를 판단하지 않는다.
- 무기록일은 미실천을 의미하지 않으며, 기록일과 실제 행동 시점이 다를 수 있다.
- 외부 제출물은 상담자가 원자료 대조, 비식별화, 사실 확인, 문장 수정 및 최종 승인을 완료해야 한다.
