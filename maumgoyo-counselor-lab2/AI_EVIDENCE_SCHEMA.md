# 마음고요 AI 표준 근거자료 v1

## 목적

`maeumgoyo_ai_evidence_v1`은 상담자용 앱의 여러 AI 출력물을 하나의 공통 근거자료로 통합하기 위한 JSON 규격이다. 이 파일은 보고서가 아니며 특정 보고서를 작성하라는 프롬프트도 포함하지 않는다. 회기준비, 상세분석, 자기성찰 보조, 주간 치료참여 및 재범방지 실천기록 등은 별도의 목적별 프롬프트가 이 파일을 읽어 생성한다.

누적 원본 CSV는 자료 보관과 재불러오기를 위해 계속 별도로 유지한다.

## 목적별 프롬프트 프로필

동일한 표준 JSON은 A 회기준비, B 상세 사례분석, C 다음 회기 임상맥락, D 자기성찰 질문, E 예비 자기성찰문, F 주간 치료참여 및 재범방지 실천기록, G 월간 치료과정 요약, H 상담자 의견서 근거개요에 공통으로 사용한다. 프로필은 자료를 바꾸지 않고 결과물의 목적과 형식만 지정한다.

## 출처 코드

| 코드 | 의미 | 외부 문서에서의 취급 |
|---|---|---|
| `SR` | 내담자 자기보고 또는 내담자 앱 입력 | 독립적으로 확인된 사실과 구분한다. |
| `AM` | 앱이 원자료에서 계산한 수치·연쇄·요약 | 상담자의 검토가 필요한 보조정보이다. |
| `CO` | 상담자가 직접 입력한 관찰·확인 사실 | 직접 확인 범위와 한계를 함께 기술한다. |
| `AI` | AI가 생성했으나 채택되지 않은 해석 | 사실처럼 사용하지 않는다. |
| `CH` | 상담자가 검토하여 채택한 AI 제안·해석 | 상담자의 판단임을 표시하고 원자료와 대조한다. |

## 최상위 구조

| 필드 | 내용 |
|---|---|
| `schema_version` | `maeumgoyo_ai_evidence_v1` |
| `package_metadata` | 패키지 ID, 앱 버전, 생성시각, 비식별 코드, 프로젝트명 |
| `intended_use` | 허용 용도, 금지된 결론, 사람의 최종 검토 의무 |
| `provenance_legend` | 출처 코드 설명 |
| `source_integrity` | 원본 파일, CSV 규격, 행 수, 제외 행, 자료형별 건수, 파싱 실패 |
| `period_and_completeness` | 최초·최종 기록일, 기록일, 관찰기간 내부 무기록일, 지연 기록 |
| `privacy` | 개인정보 형식 후보와 자동 탐지의 한계 |
| `treatment_participation` | 회기일, 참석 상태, 상담과제, 확인 한계와 상담자 근거 참조 |
| `current_period_app_measures` | 주간 수치, 위험·부분성공·실천 관련 건수, 예기불안 요약, 감사 기록(`gratitude_reflection`) |
| `comparison` | 이전 요약과 현재 수치 및 차이 |
| `practice_summary` | 실천별 계획·완료·완료율·즐거움·성취감 |
| `derived_chains` | 앱이 재구성한 연쇄와 검토 상태 |
| `daily_reflection_summaries` | 날짜별 자기관찰 요약 |
| `source_evidence` | CSV 행별 원자료와 `SR-*` 근거 ID |
| `counselor_evidence` | 상담 메모·과제·직접 확인·확인 한계와 `CO-*` 근거 ID |
| `clinical_continuity` | 이전 맥락, 채택·보류 제안, 다음 회기 질문 |
| `limitations` | 자기보고, 무기록일, 앱 계산, 외부 제출에 관한 해석 한계 |
| `validation` | 검증기 버전, 상태, 오류·경고 수, 오류·경고 내용과 개별 점검 결과 |

## 감사 기록 (gratitude_reflection)

`daily_checkin`의 `gratitude_others`(타인에 대한 감사), `gratitude_self`(자신에 대한 감사)는 `expansion_score`(삶의 경험 확장/만족도)와 같은 카드에서 나오는 자료이며, `current_period_app_measures.gratitude_reflection`(provenance `AM`)에 다음과 같이 집계된다.

| 필드 | 내용 |
|---|---|
| `others_count` / `self_count` | 이번 기간 전체 감사 건수 |
| `days_with_any_gratitude` | 감사를 1건 이상 기록한 날 수 |
| `compared_to_previous_snapshot` | 직전 회기 스냅샷 대비 건수 차이(단순 수치. 개선/악화 라벨 없음) |
| `representative_entries` | 최신순으로 선정한 대표 원문과 `SR-*` 근거 ID(타인/자신 구분) |

날짜별 원문은 `daily_reflection_summaries`에도 그날의 삶의 경험 확장/만족도 옆에 함께 표시된다.

이 항목은 선택 입력이므로 다음을 지킨다.

- 건수·존재 여부만으로 반성의 진정성, 정서 개선, 치료 성실도를 판단하지 않는다.
- `comparison`이나 회기비교 화면의 "개선/악화" 자동판정 대상에 포함하지 않는다.
- 대표 원문 선정은 최신순일 뿐이며 내용에 대한 긍정·부정 평가를 포함하지 않는다.

## 프로필 F 작성에 필요한 최소 조건

법원 제출용 「주간 치료참여 및 재범방지 실천기록」 초안을 만들기 전 다음을 확인한다.

1. `treatment_participation.session_date`와 `attendance_status`가 입력되어 있다.
2. 상담과제가 있다면 `assigned_tasks`와 `CO-TASK-*` 근거가 존재한다.
3. 상담자가 직접 확인한 사실과 내담자의 일상 자기보고가 구분되어 있다.
4. `period_and_completeness`의 무기록일과 지연 기록을 숨기지 않는다.
5. 구체적인 실천은 반드시 `SR-*` 또는 `CO-*` 근거 ID와 연결한다.
6. 계획, 실제 수행, 부분 수행, 미수행을 서로 구분한다.
7. 개인정보 후보 경고가 없더라도 실명·기관명·구체적 장소를 사람이 다시 확인한다.

## 해석 제한

- 기록의 존재나 빈도는 반성의 진정성을 증명하지 않는다.
- 수치가 개선되어도 치료효과나 재범위험 감소를 자동으로 결론내리지 않는다.
- 감사 기록(`gratitude_others`/`gratitude_self`)은 선택 입력이며, 건수·기록일 증감만으로 정서 개선이나 치료 성실도를 판단하지 않는다.
- 무기록일은 미실천을 의미하지 않으며, 기록일과 실제 행동 시점이 다를 수 있다.
- 외부 제출물은 상담자가 원자료 대조, 비식별화, 사실 확인, 문장 수정 및 최종 승인을 완료해야 한다.
