# A~H 목적별 프롬프트 운영 안내

## 공통 운영방식

1. CSV를 불러오고 상담자 확인 근거를 입력한다.
2. `AI 표준 근거자료 저장(JSON)`으로 공통 근거자료를 만든다.
3. 만들 문서의 프로필 A~H를 선택한다.
4. 선택 프롬프트를 복사하거나 MD로 저장한다.
5. AI에 JSON과 프롬프트를 함께 제공한다.
6. 결과물의 주요 사실과 근거 ID를 원자료에 대조한다.
7. 상담자가 비식별화, 사실관계, 해석의 한계와 문서 사용범위를 검토한다.

## 프로필별 용도

| 프로필 | 결과물 | 주요 사용자 | 외부 제출 |
|---|---|---|---|
| A | 50분 상담 회기준비안 | 상담자 | 아니오 |
| B | 상세 사례분석 초안 | 상담자 | 원칙적으로 아니오 |
| C | 다음 회기 임상맥락 | 상담자 | 아니오 |
| D | 자기성찰 질문 | 상담자·내담자 | 아니오 |
| E | 예비 자기성찰문 | 내담자 작성 보조 | 그대로 제출 금지 |
| F | 주간 치료참여 및 재범방지 실천기록 | 상담자 검토 후 사용 | 최종 승인 후 가능 |
| G | 월간 치료과정 요약 | 상담자 | 용도에 따라 승인 필요 |
| H | 상담자 의견서 근거개요 | 상담자 | 개요 자체는 제출 금지 |

## 단일 근거자료 원칙

- AI 입력용 근거자료는 표준 JSON 하나를 기준으로 한다.
- 프롬프트는 자료를 다시 추출하거나 변형하지 않고 결과물의 목적과 형식만 지정한다.
- 새로운 문서가 필요하면 CSV 출력 기능을 추가하기보다 새 프롬프트 프로필을 추가한다.
- 원본 CSV는 보관·재분석 용도로 별도 유지한다.
- 상담기록·비교 요약 JSON은 앱 내부의 다음 회기 수치 비교용으로 유지한다.

## 공통 금지사항

- 기록에 없는 사실이나 감정 생성
- 자기보고와 상담자 확인의 혼합
- 반성의 진정성·완치·재범위험 감소 판단
- 공식 위험평가 없이 위험등급 부여
- 선처·양형·형량에 관한 의견 생성
- 피해자나 제3자의 식별정보 포함
- AI 초안을 상담자 검토 없이 외부 제출

## 이전 방식 출력

회기준비 MD, 회기맥락 MD, 상세분석 TXT는 기존 자료 및 작업 흐름과의 호환을 위해 남겨 두었다. 신규 사례와 신규 회기에서는 표준 JSON과 목적별 프로필을 우선 사용한다.
